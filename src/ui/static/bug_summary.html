Bug Summary Analyzed 2m ago

Infinite redirect loop on OAuth callback for regional AWS instances

The authentication service fails to handshake correctly when the user is routed through the 'us-west-2' gateway, resulting in a 302 loop between the login portal and the regional balancer.

rebase_edit Suggested Reproduction Steps

1

Configure VPN

Route traffic through a US-West-2 exit node to simulate regional access.

2

Initiate OAuth

Navigate to /auth/login and select 'AWS Single Sign-On' as the provider.

3

Monitor Network

Open DevTools and watch for cyclic 302 responses between domains.

Expected Results

User should be redirected to the regional AWS console dashboard after successful authentication, with a valid session token corresponding to the 'us-west-2' region.

Actual Results

[14:22:01] GET https://auth.acme.com/callback?code=... (302)
[14:22:01] GET https://us-west-2.console.acme.com/login (302)
[14:22:02] GET https://auth.acme.com/callback?code=... (302)
... (Loop detected after 15 redirections)

smart_toy Triage Recommendation

This issue should be treated as a Blocker for the upcoming regional expansion. The root cause appears to be a mismatch in the callback URL whitelist parameters specific to AWS edge locations. Recommended fix involves updating the auth-config.yaml to support wildcards for regional subdomains.
